Introduction

This is a coding project developed for RideCell by Gary Guo on October 23rd, 2016.

Architect and implement the backend for a street parking reservation system for mobile apps to use. You can use any language or platform. Support the following features:

  • User registration & login: New users should be able to create a new account using their email address, set up a password, and set up their phone number.
    • Bonus: Integrate with an OAuth service and use that for registration and authentication.
    • Bonus: Accept credit cards for payments (feel free to use third party libraries like Stripe or Braintree)
    • Bonus: Validate that phone numbers are real (feel free to use third party libraries like Twilio or Plivo)
  • Parking spot viewing & search interface: Provide APIs for users to be able to see available parking spots on a map, search for an address and find nearby unreserved parking, and see how much a particular parking spot costs.
    • Bonus: Enforce a maximum/minimum length of time that the parking spot can be reserved
  • Parking spot reservation: Users should be able to reserve a parking spot, view their reservation, and cancel it. Note that reservation should start immediately.

Endpoints

The following links provide boiler plate UI for the REST endpoints.

  • Users Endpoint: This endpoint enables creation, access_token retrieval and user profile retrieval.
  • Parking Endpoint: This endpoint filtering of parking locations.
  • Reservations Endpoint: This endpoint enables creation, retrieval, update and deletion reservations.
